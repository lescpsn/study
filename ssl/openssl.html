git clone https://github.com/openssl/openssl.git OpenSSL

2 编译

2.1 Windows

  1. 安装vs community 2013
    • 下载
    注意: vs2015及其以上的版本可能有坑, 不适用本文档, 暂不想花时间去解决, 直接先用vs2013版本

官网https://www.visualstudio.com/downloads/下载vs2013的iso版本
    • 安装
    指定好安装目录 -> 自定义安装 -> 勾选VC++ -> 一路回车法
    • 修改PATH环境变量(可选)
    vs安装路径下的C:\Local\Microsoft Visual Studio 12.0\VC\bin\cl.exe即编译器, 相当于linux下的gcc

将C:\Local\Microsoft Visual Studio 12.0\VC\bin路径添加到环境变量PATH中
    • 添加INCLUDE环境变量(可选)
    INCLUDE环境变量相当于头文件的搜索路径, 将

C:\Local\Microsoft Visual Studio 12.0\VC\include

C:\Program Files (x86)\Microsoft SDKs\Windows\v7.1A\Include

添加到环境变量INCLUDE中
    • 添加LIB环境变量(可选)
    LIB环境变量相当于库文件链接搜索路径, 将

C:\Local\Microsoft Visual Studio 12.0\VC\lib

C:\Program Files (x86)\Microsoft SDKs\Windows\v7.1A\Lib

添加到环境变量LIB中
  2. 安装perl
    • 下载
    官网https://www.perl.org/get.html下载 ActiveState Perl对应系统版本(32位或者64位)
    • 安装及配置
    选择custom方式 -> 指定好安装目录 -> 一路回车法
    • 测试安装是否成功
    where perl
  3. 安装nasm(可选)
    vs自带的汇编编译器，OpenSSL不支持，需要安装另外的nasm汇编编译器

官网 http://www.nasm.us/pub/nasm/releasebuilds/2.12.02/win64/

下载对应的windows版本
    • 安装
    指定好安装目录 -> 一路回车法, 之后将安装后的目录添加到环境变量PATH中去。
  4. 编译OpenSSL
    cd OpenSSL
%comspec% /k ""C:\Local\Microsoft Visual Studio 12.0\VC\vcvarsall.bat"" x86  (这步很重要)
perl Configure VC-WIN32 --prefix="C:\Local\usr\OpenSSL32"
nmake
nmake test
  5. 生成
    头文件        : ./include/openssl
动态库引用文件 : *.lib            
动态库        : *.dll

2.2 Linux

  • 下载
官网https://www.openssl.org/下载最新稳定版本
  • 编译
进入vs的安装目录下 D:\Program Files\Microsoft Visual Studio 12.0\Common7\Tools\Shortcuts
注意：一定右键以管理员方式 运行 VS2013 x64 本机工具命令提示，而不能以cmd行运行。
（若以cmd行运行，则环境变量的信息没有带过来）
nmake clean
perl Configure VC-WIN64A --prefix=C:\Local\usr\openssl64

网络上很多文章描述的还需要ms\do_nasm这一步这种方式已经过时了，在openssl最新的1.1以上的版本是不需要的的，直接nmake就可以了。

nmake

nmake test

nmake install

3 Linux下编译安装

3.1 下载源代码

git clone https://github.com/openssl/openssl.git

3.2 编译源代码

./config
make
make test
sudo make install

3.3 修改环境变量

ldd /usr/local/lib/libssl.so
➜  ~ ldd /usr/local/lib/libssl.so
        linux-vdso.so.1 =>  (0x00007fff46c16000)
        libcrypto.so.1.1 => not found
export LD_LIBRARY_PATH=/usr/local/lib //否则libcrypto.so.1.1 => not found
